Phase 1: Backend Architecture & Implementation Plan1. Project Setup & StandardFramework: Node.js with Express.js and TypeScript.Architecture: Use a Service-Based Architecture.controllers/: Handles HTTP requests and responses.services/: Contains core business logic (Inventory, Billing, etc.).routes/: Defines API endpoints.middleware/: Auth guards and error handling.ORM: Prisma for type-safe database interactions with Supabase.2. Database & Schema InitializationRun the Prisma schema provided previously to create the following tables:User Management: Roles and Permissions (RBAC) to handle different access levels for Admins, Doctors, and Nurses.Inventory System: inventory_batches with location_type (Warehouse, Vehicle, Nurse) to manage the three-tier stock flow.Patient & Visit: Tables for Patients, Bookings, and VisitRecords.3. Core Business Services (The Logic)Implement these specific services:InventoryService:Stock Transfer: Logic to move stock from Warehouse to a Nurse's individual mobile kit.Dispense Medicine: Logic to deduct stock from the specific Nurse’s batch during a home visit.BillingService:Automatically calculate consultation_fee (from doctor profile) + travel_cost + medicine_cost.Generate a unique Invoice record.FleetService:Assign MedicalTeams and Vehicles to specific Bookings.4. API Endpoint SecuritySupabase Auth Integration: Verify JWT tokens from Supabase on every request.RBAC Middleware: Ensure only Admins can perform stock transfers and only Nurses/Doctors can record visit notes.
