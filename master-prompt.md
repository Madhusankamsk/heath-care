Project Master Plan: Mobile Healthcare & Fleet Management PWAContext & Technology StackBuild a Mobile Healthcare & Fleet Management System as a Progressive Web App (PWA).Frontend: React.js (Vite) + Tailwind CSS + Lucide React (Icons).Backend: Node.js (Express) + TypeScript.ORM: Prisma.Database & Auth: Supabase (PostgreSQL).Architecture: Layered Architecture (Controllers, Services, Routes).1. Database Schema (Prisma)Initialize the database with the following core entities based on the technical proposal:Profiles & RBAC: Roles (Admin, Doctor, Nurse, Driver) with granular permissions. Includes base_consultation_fee for staff.Fleet: Vehicles and MedicalTeams (Supports multiple nurses, with or without doctors).Inventory (Three-Tier):Main Warehouse: Central stock.Mobile Kits: Inventory assigned to specific Nurses or Vehicles.Batches: Tracks buying_price, selling_price, batch_no, and expiry_date.Patient & Visits: Patients, Bookings, and VisitRecords for clinical notes and vitals.Billing: Invoices linked to bookings or OPD, calculating consultation, travel, and medicine costs.2. Core Logic RequirementsStock Transfer Flow: Implement a service to transfer medicine from Warehouse to a Nurse's Profile ID (Mobile Kit).Dispensing Logic: When a Nurse records medicine usage during a visit, the system must deduct quantity from that specific Nurse’s InventoryBatch.Offline Sync: Implement a sync mechanism to allow staff to record visit data and medicine usage in low-network areas, syncing to Supabase when online.Fleet Dispatch: Logic to assign a MedicalTeam and Vehicle to a Booking and track status (Pending, Dispatched, Ongoing, Completed).3. Milestone Implementation PhasesPhase 1 (Core Operations): Admin Dashboard, Staff PWA, Patient Booking, and Basic Billing.Phase 2 (Advanced Inventory): Multi-location stock management (Warehouse to Nurse) and Low Stock/Expiry Alerts.Phase 3 (OPD Module): Walk-in patient queue, token system, and clinic-based dispensing.4. Instructions for Cursor.aiInitialize Project: Create the folder structure (client/server).Prisma Setup: Generate the schema.prisma file based on the entities above.API Development: Create CRUD for Patients, Staff, and Inventory.Business Services: Write the logic for stockTransferService and billingService.PWA Setup: Configure Vite PWA plugin for manifest and service worker.
